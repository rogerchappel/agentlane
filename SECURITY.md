# Security Policy

## Supported Versions

`agentlane` is pre-1.0 today. Only the latest `main` branch state is considered supported until tagged releases begin.

| Version | Supported |
| --- | --- |
| main | Yes |
| tagged releases before 1.0 | Best effort |
| archived commits | No |

## Reporting a Vulnerability

Please do not report suspected vulnerabilities in public issues, pull requests, or discussions.

Email the maintainer privately or open a low-detail GitHub issue asking for a private reporting path. Do not include exploit details, secrets, personal data, or sensitive technical details in public messages.

## What to Include

When a private reporting path is available, include:

- a clear description of the issue
- affected versions, files, packages, workflows, or configuration
- steps to reproduce or an attack scenario when safe to share
- potential impact
- suggested mitigation, if known

## Response Expectations

Maintainers review good-faith reports as capacity allows. This project does not currently offer paid support, SLAs, or guaranteed patch timelines.

## Scope

In scope:

- vulnerabilities in `agentlane`
- insecure default output or safety guidance shipped by this project
- CI, release, or dependency guidance maintained by this project

Out of scope:

- general support requests
- guaranteed maintenance timelines
- issues in unrelated downstream projects

## Disclosure

Coordinate disclosure with maintainers before publishing vulnerability details.
